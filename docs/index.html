Study overview

Why preserve competing hypotheses?

Remote photoplethysmography pipelines commonly collapse a video window into one heart rate even when several hypotheses remain plausible. This makes it difficult to distinguish candidate absence from an arbitration error or an unresolved source conflict.

VitalsSight preserves source identity, ranks observed hypotheses using prediction-time evidence and reports the selected value separately from a proposed release or review state. In participant-disjoint development, the joint candidate path achieved a participant-equal mean absolute error of 1.835 beats per minute in 42 participants. A frozen base selector was then evaluated once in 83 untouched participants and 2,120 windows, achieving 4.217 beats per minute. It outperformed single-route TS-CAN but did not differ from matched ExtraTrees. The separately frozen release gate transported poorly, identifying cross-domain score shift rather than loss of selector output.

Claim boundary

The work supports traceable candidate-preserving inference. It does not establish calibrated clinical safety, autonomous release, real-time clinical utility or medical-device performance.

Method

One inference object, four explicit outputs

Video evidence forms a variable-size constellation. A causal path selects an observed candidate; the selected heart rate, evidence packet, research score and proposed state remain separate.

01

Construct

Regional, classical, learned, transformer and optional correction routes contribute observed hypotheses with retained source identity.

02

Relate

Candidate-intrinsic evidence is evaluated together with agreement, harmonic structure, temporal support and source relations.

03

Select

A causal joint path returns one hypothesis that was actually observed instead of synthesizing an untraceable average.

04

Report

The selected value and evidence packet remain linked to a proposed release or review state without becoming a safety guarantee.

Evidence hierarchy

Internal improvement and external transport are reported separately

Participants, not video windows, anchor inference. The external evaluation was frozen before reference access; later refinements remain explicitly retrospective.

Research software

Evidence operations, not a bare prediction endpoint

The reference implementation connects quality qualification, candidate evidence, release/review/retake states, audit records, report exports and an evidence-bounded local assistant.

Availability

Reproducibility with explicit data boundaries

The public repository contains the research implementation, protocol descriptors, aggregate manuscript metrics and the exact submission-facing reproducibility package. It intentionally excludes raw videos, identifiable participant frames, provider-controlled labels, credentials and private execution logs.
