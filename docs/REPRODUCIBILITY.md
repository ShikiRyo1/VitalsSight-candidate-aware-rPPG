# Reproducibility map

## Analysis order

1. Generate route- and ROI-specific heart-rate candidates.
2. Preserve candidate provenance, quality, harmonic and agreement evidence.
3. Fit or apply the candidate selector using participant-disjoint partitions.
4. Evaluate all-output accuracy separately from release/review behavior.
5. Resample whole participants for cluster-aware uncertainty.
6. Treat subject-disjoint risk analysis as a separate, explicitly bounded experiment.
7. Report each dataset, protocol key and statistical unit separately.

## Fixed seeds and thresholds

- Model seeds: `704`, `1704`, `2704`.
- Split/bootstrap seeds: `2024`, `2025`, `2026`.
- Analytical error thresholds: `5`, `8`, `10`, `15` BPM.
- Reference heart rate and candidate absolute error are training/evaluation fields, never release-time inference features.

## Entry points

- `scripts/run_t467_ubfc_protocol_aligned_candidate_selector.py`: protocol-aligned primary candidate table.
- `scripts/run_t730_harmonic_aware_selector.py`: harmonic-aware selector.
- `scripts/run_t731_candidate_ranker_harmonic_rescue.py`: candidate-ranker rescue branch.
- `scripts/run_t732_t731_bootstrap_ablation.py`: selector ablation and bootstrap summaries.
- `scripts/run_t901_subject_cluster_bootstrap.py`: participant-equal endpoints and whole-participant resampling.
- `scripts/run_t902_subject_disjoint_risk_control.py`: calibration/test participant split and risk diagnostics.
- `scripts/run_t913_isolated_runtime_profile.py`: measured partial runtime path; deep predictions may be precomputed.

Several scripts consume intermediate CSV files generated by preceding entry points. They fail closed when inputs are missing rather than silently substituting values. The public repository deliberately excludes those large or participant-linked intermediates; aggregate manuscript values are preserved in `reproducibility/headline_metrics.csv`.

## Claim boundary

The repository supports inspection of the candidate-aware architecture, executed analysis logic and aggregate protocol outputs. It does not establish matched literature-level superiority, participant-level risk guarantees, external generalisation, fairness, clinical utility or end-to-end real-time readiness.
